class=text><p>Python provides inbuilt functions for creating, writing, and reading files. There are two types of files that can be handled in python, normal text files and binary files (written in binary language, 0s, and 1s).</p><ul><li><strong>Text files: </strong>In this type of file, Each line of text is terminated with a special character called EOL (End of Line), which is the new line character (&#8216;\\n&#8217;) in python by default.</li><li><strong>Binary files: </strong>In this type of file, there is no terminator for a line, and the data is stored after converting it into machine-understandable binary language.</li></ul><p>In this article, we will be focusing on opening, closing, reading, and writing data in a text file.</p><div id=GFG_AD_gfg_mobile_336x280></div><p style=text-align:center><strong>File Access Modes</strong></p><p>Access modes govern the type of operations possible in the opened file. It refers to how the file will be used once its opened. These modes also define the location of the <strong>File Handle</strong> in the file. File handle is like a cursor, which defines from where the data has to be read or written in the file. There are 6 access modes in python.</p><div id=_GFG_ABP_Incontent_728x90 style=text-align:center></div><div style="min-height:280px;text-align:center;margin:20px 0" id=GFG_AD_InContent_Desktop_728x280></div><ol><li><strong>Read Only (&#8216;r&#8217;) : </strong>Open text file for reading. The handle is positioned at the beginning of the file. If the file does not exists, raises the I/O error. This is also the default mode in which a file is opened.</li><li><strong>Read and Write (&#8216;r+&#8217;):</strong> Open the file for reading and writing. The handle is positioned at the beginning of the file. Raises I/O error if the file does not exist.</li><li><strong>Write Only (&#8216;w&#8217;) :</strong> Open the file for writing. For the existing files, the data is truncated and over-written. The handle is positioned at the beginning of the file. Creates the file if the file does not exist.</li><li><strong>Write and Read (&#8216;w+&#8217;) </strong>: Open the file for reading and writing. For an existing file, data is truncated and over-written. The handle is positioned at the beginning of the file.</li><li><strong>Append Only (&#8216;a&#8217;)</strong>: Open the file for writing. The file is created if it does not exist. The handle is positioned at the end of the file. The data being written will be inserted at the end, after the existing data.</li><li><strong>Append and Read (&#8216;a+&#8217;) : </strong>Open the file for reading and writing. The file is created if it does not exist. The handle is positioned at the end of the file. The data being written will be inserted at the end, after the existing data.</li></ol><p style=text-align:center><strong>How Files are Loaded into Primary Memory</strong></p><p>There are two kinds of memory in a computer i.e. Primary and Secondary memory every file that you saved or anyone saved is on secondary memory cause any data in primary memory is deleted when the computer is powered off. So when you need to change any text file or just to work with them in python you need to load that file into primary memory. Python interacts with files loaded in primary memory or main memory through <strong>&#8220;file handlers&#8221; </strong>( This is how your operating system gives access to python to interact with the file you opened by searching the file in its memory if found it returns a file handler and then you can work with the file ).</p><p style=text-align:center><strong>Opening a File</strong></p><div id=GFG_AD_Desktop_InContent_ATF_336x280 style=text-align:center></div><p>It is done using the open() function. No module is required to be imported for this function.</p><pre>File_object = open(r"File_Name","Access_Mode")</pre><p>The file should exist in the same directory as the python program file else, the full address of the file should be written in place of the filename. Note: The <strong>r</strong> is placed before the filename to prevent the characters in the filename string to be treated as special characters. For example, if there is \\temp in the file address, then \\t is treated as the tab character, and an error is raised of invalid address. The r makes the string raw, that is, it tells that the string is without any special characters. The r can be ignored if the file is in the same directory and the address is not being placed.&nbsp;</p><div class=noIdeBtnDiv><div class=responsive-tabs><h2 class=tabtitle>Python</h2><div
